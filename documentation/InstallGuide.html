  1. Partioning and Formatting

Partitioning and Formatting

    1. Brief introduction to partioning and explaination of how best to partition one drive

Partitioning is breaking the hard drive into logical parts to store relavent information for the system. Historically this was done for many reasons but has origins in older tape systems and distributed network systems. It is a great tool for being able to manage redundacy and temporary data. For most home workstation systems the partitiion is very simple. We generally need only create three partitions: A small boot partition and the start of the hard drive, next we’ll have some temporary data storage and lastly the remainder of the system.

Breaking the data into three (or more) logical partitions has many advantages (and disadvantages) so I’ll only cover them briefly. Having a small boot partion at the start of the drive is the obvious place to have bootable files, you could really store them anywhere however the position of the small partition will make sense shortly. We keep only the files necessary to boot into the system in this space. As they are not needed once the system is up, the partition can then be unmounted. Common files to find here are the Kernel image itself aswell as system maps and boot configuration files. We recommend having at least 10MB (Space to keep at least one kernel and one backup) and not needing any more than 50MB.

The temporary data storage is called swap space. The swap space is similar to a windows or MacOS virtual memory file. The advantages of having a fixed partition for this information at the top of a hard drive is that you ensure speed of access. The obvious drawback however is that it is of a fixed maximum size (This can also been seen as an advantage considering some algorithms for handling virtual memory allocation and release). Swap space can be a difficult one to allocate, the general rule of one – three  times your physical memory may no longer apply. The rest of your disk should be for the rest of the system. If you want to learn more or need a more complex setup click here.

    1. Brief explaination of formatting and filesystems (May need help with this section)

Now that we have partitions created we need to format the filesystem so as we can write to it. Linux can natively read and write to many different filesystem types. Most commonly found is the second extended filesystem which is a robust filesystem designed to recoverable and extesible. Like any good movie series though there is a third iteration, but unlike the movies et3fs is an improvement on the last by adding journaling support for the volume.
ReiserFS is a common alternative to the mainstream filesystems. It is a journaling filesystem which has been designed with small files in mind, but that's not to say that it doesn't handle large files well. As the homepage suggests comparitive tests show ReiserFS to be more efficient than its ext3fs conterpart.
To further understand the complexities of filesystems my first stop would be the HOWTO guide on The Linux Documentation Project. In summary, we recommend formating with ext3fs as it's vanilla yet has the support for journaling which improves stability.
